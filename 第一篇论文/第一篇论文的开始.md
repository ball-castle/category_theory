## 结论

目前最适合你的方向不是直接攻 ((\infty,1))-范畴模型或完全弱双范畴，而是把维度控制在 **2-范畴／双范畴层面**：

> **单值 framed bicategories（或 proarrow equipments）的理论与机器形式化**

建议论文题目暂定为：

> **Univalent Framed Bicategories in UniMath: Companions, Conjoints, Base Change, and Mates**
> **单值 framed bicategories 的机器形式化：companions、conjoints、基变换与 mates 演算**

这是我认为目前“前沿性、可执行性、论文体量和二区可能性”平衡最好的一条线。

---

## 一、它具体研究什么

一个 framed bicategory 可以粗略理解为一种双范畴：

* 一类箭头表示普通映射，例如函数、函子、环同态；
* 另一类箭头表示广义关系，例如关系、profunctor、双模、span；
* 每个普通映射都能诱导相应的广义关系，即 companion 和 conjoint；
* 因而可以统一描述拉回、推前、基变换和 mates correspondence。

典型例子包括：

[
(\mathrm{Set},\text{functions},\text{relations}),
]

[
(\mathrm{Cat},\text{functors},\text{profunctors}),
]

[
(\mathrm{Ring},\text{homomorphisms},\text{bimodules}).
]

原始 framed bicategory 理论正是为了把“普通映射”和“广义态射”同时放入一个具有良好基变换结构的高维范畴框架中。([arXiv][1])

---

## 二、为什么选它，而不选更大的未解问题

### 1. 它是明确存在的文献空缺

2024 年的《Univalent Double Categories》已经在 UniMath 中形式化了单值双范畴，但论文结论明确写道：

* 当前形式化尚未包含 framed bicategories；
* 可以通过构造单值双范畴的一个满子双范畴，得到单值 framed bicategories 的双范畴；
* monoidal double categories、framed bicategories 和更弱的 double bicategories 都是后续方向。([arXiv][2])

这意味着你不是自己凭空制造一个题目，而是在一篇成熟工作的结尾直接接续一个尚未完成的模块。

根据截至 2026 年 7 月的公开检索，我没有找到一篇已经专门完成“univalent framed bicategories in UniMath”这一整套工作的论文。不过，代码库中可能存在尚未成文或尚未被搜索引擎收录的开发，因此正式立项前必须向 UniMath 社区确认。

### 2. 近期理论进展正好提供了新工具

2026 年更新的 doubly weak double categories 工作重新整理了完全弱双范畴、proarrow equipments、companions、conjoints 和 mates correspondence 之间的关系。文中说明，equipment 可以通过“每个适当态射都具有 companion 和 conjoint”来刻画，并在更弱的双范畴框架下表达 bicategorical mates correspondence。([arXiv][3])

你不需要把整篇完全弱双范畴理论形式化，而可以利用它提供的语言和结果，完成较小的 framed bicategory 子问题。

### 3. 它是真正的高维范畴论，但不涉及无限层相干

这个题目需要处理：

* 对象；
* 两类 1-态射；
* 方形 2-态射；
* pseudofunctor；
* associator 和 unitor；
* companion/conjoint；
* 双范畴中的伴随和 mates。

所以它确实属于高维范畴论，而不是普通范畴论换一个名称。

但它仍停留在有限的二维相干层面，不要求你从头掌握：

* quasi-category；
* complete Segal space；
* model category；
* simplicial localization；
* 无限层 horn filling；
* ((\infty,n))-范畴模型比较。

目前 Rzk 已经形式化了 (\infty)-范畴 Yoneda 引理，但这仍是专用类型论和专用证明助手上的前沿工作，直接从这里开始风险明显更高。([arXiv][4])

## 三、建议的精确研究问题

不要把题目写成泛泛的“形式化 framed bicategories”。建议围绕以下四个主定理组织。

### 定理组 A：companions 和 conjoints 的单值理论

在一个单值 pseudo double category (\mathbb D) 中：

1. 定义一个 vertical morphism 的 companion；
2. 定义 conjoint；
3. 证明两个 companion 之间存在规范的 globular isomorphism；
4. 研究在单值条件下，companion 的选择空间是否为命题或可缩类型；
5. 证明 companion 和 conjoint 在复合与恒等态射下封闭。

这一部分的核心价值是把“选择 companion”尽量转化为一种与具体选择无关的性质。

### 定理组 B：framed bicategory 的两种刻画

比较以下两个定义：

[
\text{每个 vertical morphism 都有 companion 和 conjoint}
]

与

[
(s,t):\mathbb D_1\longrightarrow
\mathbb D_0\times\mathbb D_0
]

满足适当的 fibration/opfibration 条件。

目标是机器验证二者在合适假设下等价。原始 framed bicategory 理论正是通过 categorical fibrations 避免大量手写相干条件。([arXiv][1])

这会是论文中最像“新数学定理”的部分，而不只是代码工程。

### 定理组 C：单值 framed bicategories 的双范畴

构造：

[
\mathbf{FramedDblCat}_{\mathrm{univ}}
]

其：

* 0-胞是单值 framed bicategories；
* 1-胞是保存 framed 结构的 lax/strong double functors；
* 2-胞是适当的 double transformations。

然后证明：

> (\mathbf{FramedDblCat}_{\mathrm{univ}}) 本身是一个单值 bicategory。

基本策略是使用已有的单值双范畴双范畴，构造一个满足 framed 性质的 full subbicategory。这个方案正是已有论文明确提出的后续路径。

### 定理组 D：基变换与 mates 演算

形式化以下结构：

* restriction 和 extension；
* companion/conjoint 诱导的 base-change；
* mate of a square；
* mates 对水平、垂直复合的函子性；
* Beck–Chevalley 型条件；
* adjunction 与 double categorical mate 之间的对应。

这一部分能够显著提高论文质量，因为它展示 framed 结构不只是一个定义，而是能够真正简化复杂的二维图表推理。

## 四、最小可发表版与二区增强版

### 最小可发表版本

至少需要完成：

1. framed bicategory 的 UniMath 定义；
2. companion/conjoint 的唯一性理论；
3. framed 性质与 fibration 性质的关系；
4. 单值 framed bicategories 组成单值 bicategory；
5. 两个非平凡例子；
6. 所有结果机器验证，并形成可复用库。

这个版本更像 Applied Categorical Structures 或偏专门的范畴论／形式化期刊论文。

### 冲击二区附近的增强版本

在以上基础上再加入：

1. 完整的 mates correspondence 库；
2. 一套自动化 tactic，自动化处理 unitor、associator、whiskering 和 mate 转换；
3. 对形式化前后证明长度进行实验比较；
4. 至少三个实例：

   * spans；
   * cospans；
   * relations 或 profunctors；
5. 代码进入 UniMath 主库，或者至少形成独立、可复现、文档完整的库。

形式化基础设施不足确实是当前高阶数学机器验证的重要瓶颈。2025 年 Lean 中对 (\mathbf{Cat}) 的余极限形式化，就把自身定位为未来形式化 (\infty)-范畴论所需的前置基础。([arXiv][5])

如果只有“定义若干结构并验证若干显然引理”，论文强度通常不够。必须至少具有一项：

[
\text{新结构定理}
\quad+\quad
\text{大规模形式化}
\quad+\quad
\text{自动化或重要实例}.
]

## 五、推荐实例的选择

建议按照以下顺序做。

### 第一个实例：Span

设 (\mathcal C) 具有拉回。构造双范畴：

* 对象：(\mathcal C) 的对象；
* vertical morphisms：(\mathcal C) 中的态射；
* horizontal morphisms：span；
* squares：span 之间的映射。

它结构清楚，适合作为第一个完整实例。

### 第二个实例：Cospan

在具有推出的范畴中构造 cospan 双范畴。

它可以检验你设计的定义是否具有良好的对偶性，而不是只适用于一个方向。

### 第三个实例：Relations 或 Profunctors

这一部分用于增强论文影响力。

其中 Prof 是最经典的 equipment，但它涉及：

* 大小问题；
* 范畴的单值化；
* profunctor 的复合；
* coend；
* 可能的宇宙层级问题。

因此把 Prof 放在论文后半部分或作为增强目标，不要作为第一个开发对象。

## 六、技术路线建议

证明助手建议优先选 **Rocq/Coq + UniMath**，而不是一开始使用 Lean。

原因不是 Lean 不好，而是：

* 现成的 univalent bicategory；
* displayed bicategory；
* two-sided displayed category；
* univalent double category；

目前已经存在于 UniMath 体系中。UniMath 本身仍在持续维护，并为单值数学提供专门基础设施。([GitHub][6])

使用 Lean 意味着你可能需要先重建大量 bicategory 和 double category 基础。除非论文的主要创新就是“在 Lean 中建立首个通用双范畴库”，否则工作量会显著扩大。

## 七、难度评价

我对这个题目的评价是：

| 维度       |        评价 |
| -------- | --------: |
| 数学难度     |      7/10 |
| 形式化难度    |    7.5/10 |
| 新颖性      |      8/10 |
| 撞题风险     |      5/10 |
| 一年内完成可能性 |      中等偏高 |
| 二区论文潜力   | 有，但依赖增强模块 |

与其他方向比较：

* 完整 ((\infty,1))-范畴形式化：9.5/10；
* doubly weak double categories 全形式化：9/10；
* 稳定 (\infty)-范畴增强唯一性：9.5/10；
* monoidal double categories 的完整单值化：8/10；
* 本题：约 7–7.5/10。

它并不是“简单题”。对于没有范畴论和证明助手经验的人，仍然需要较长准备。但在目标是高维范畴论且希望形成正式论文的前提下，它已经属于风险较可控的一档。

## 九、投稿定位

### 偏形式化与自动推理

**Journal of Automated Reasoning**

它的官方范围包括由计算机进行逻辑推理的理论、实现和应用。要投这里，论文必须突出形式化方法、自动化和可复用软件，而不能只有范畴论结果。([Springer][7])

### 偏范畴论与计算机科学交叉

**Mathematical Structures in Computer Science**

该刊强调利用数学结构推进计算机科学理论和实践，单值 bicategory、类型论语义和形式化范畴论与其范围相符。([Cambridge University Press & Assessment][8])

### 偏纯范畴论

**Applied Categorical Structures**

其范围包括范畴论在代数、几何、拓扑、物理和计算机科学中的使用，framed bicategory、equipment 和 base-change 理论属于其自然范围。([Springer][9])

需要特别说明：**SCI/JCR 分区会随年份和学科分类变化**。高水平专业范畴论期刊并不一定在行政分区上是二区。如果“必须 SCI 二区”是硬指标，论文应强化：

[
\text{机器形式化}
+
\text{自动化工具}
+
\text{实验评估},
]

而不能只做纯范畴论。

## 最终判断

这个方向的优势是：

[
\boxed{
\text{高维范畴论的真实问题}
+
\text{已有基础库}
+
\text{明确文献空缺}
+
\text{可拆解的定理包}
+
\text{形式化论文出口}
}
]

不建议你把第一篇高维范畴论文定为“形式化所有弱双范畴”或“建立一个新的 ((\infty,1))-范畴模型”。更合理的策略是先完成：

> **单值 framed bicategories + companions/conjoints + mates + UniMath**

在这一题上，达到普通专业期刊论文是现实目标；加入完整的基变换理论、自动化和多个重要实例后，才具备比较可信的 SCI 二区附近竞争力。
